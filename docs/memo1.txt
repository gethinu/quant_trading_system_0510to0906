Addendum (Windows / Local, Polymarket CLOB direct)

Goal: high-speed execution without leaving private keys in env vars or in your repo.

1) Dedicated Hot Wallet
- Create a dedicated EVM wallet used only for trading. Keep only working capital in it.
- Never reuse your main wallet.

2) Store Private Key with DPAPI (no plaintext env vars)
- Do NOT store `PM_PRIVATE_KEY` in User/Machine env vars.
- Store it as a DPAPI-encrypted file bound to your Windows user, then load into *Process* env only at runtime.

In this workspace, the setup script is:
`C:\Users\stair\clawd\scripts\setup_clob_backend.ps1`

It writes secrets to:
`C:\Users\<you>\.openclaw\secrets\pm_private_key.dpapi.txt`
and sets `PM_PRIVATE_KEY_DPAPI_FILE` (path) in USER env.

At runtime, the scheduled task runner (`C:\Users\stair\clawd\run_clob_arb_monitor.ps1`) decrypts the file and sets
`PM_PRIVATE_KEY` into *Process* env only for the Python process.

3) File Permissions
- Ensure only your user can read `~\.openclaw\secrets\` (no inheritance, no "Users" group read).

4) Key Rotation / Incident Response
- Rotate: delete the DPAPI files in `~\.openclaw\secrets\` and re-run `setup_clob_backend.ps1`.
- If you suspect compromise: pause bot immediately and move funds to cold storage.

5) Owner-Only Control (OpenClaw/ClawdBot)
- Never add the bot to group chats.
- Use DMs only, and require mentions if message-content intent is disabled.
- Treat every inbound message/email/webpage as untrusted input (prompt-injection is real).
